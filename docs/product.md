# Okane 业务产品手册 (Product Manual)

## 1. 项目愿景与定位

**Okane** 是一套以**稳健、可靠、快速**为核心目标的量化策略执行与回测平台。

### 1.1 核心价值
- **极致的稳定性**：通过严谨的工程实现，确保在复杂市场环境下指令下达的准确性与系统的持续可用。
- **高性能执行响应**：针对金融交易优化，实现低时延的行情处理与订单决策。
- **安全的策略隔离**：为策略运行提供独立的保护机制，防止策略故障或异常对交易系统的核心运行造成威胁。
- **辅助 AI 增强**：在系统稳健运行的基础上，支持接入智能决策模块，辅助进行自动化账户管理与市场分析。

## 2. 核心业务能力

- **全链条量化支持**：
    - **历史回测**：支持基于历史精准数据的策略逻辑验证，运行结果沉淀为可追溯的回测结果与摘要报告。
    - **实时模拟盘交易**：基于真实行情驱动策略运行，使用本地撮合能力完成模拟自动交易。
    - **平台自动交易**：当逻辑交易账号挂接外部平台后，策略可通过标准接口执行自动化交易。
    - **策略信号运行**：策略也可以只消费真实行情并输出信号、日志和通知。
- **策略、交易环境、运行结果分离模型**：平台将策略逻辑、逻辑交易账号和策略运行结果作为三类独立业务对象进行管理。
- **统一逻辑交易账号体系**：逻辑交易账号作为可复用的交易环境与账本主体，统一承载资金规则、撮合规则和结算语义。
- **多账号多后端管理**：支持同时管理多个独立的逻辑交易账号，并按账号后端区分本地撮合与外部平台执行。
- **统一的市场中台**：屏蔽不同市场间的数据差异，提供标准化、高可用的行情接入与分发能力。

## 3. 业务定义

### 3.1 逻辑交易账号

- **逻辑交易账号**是平台内部统一的交易环境与账本实体。
- 逻辑交易账号负责定义初始资金、手续费、滑点、撮合规则和结算语义。
- 逻辑交易账号负责承载运行过程中的订单、成交、持仓和资金快照。
- 逻辑交易账号以可复用的方式服务于策略运行，不表达策略版本，也不表达某次运行结果本身。

### 3.2 策略与运行结果

- **策略**用于承载交易逻辑、当前草稿和参数定义。
- **策略运行结果**用于承载某个源码状态、参数状态、逻辑交易账号和运行模式组合下的执行产物。
- 策略运行结果负责记录收益、回撤、订单、成交、日志、信号和报告摘要。
- 平台通过运行结果来组织测试历史和归档历史，而不是通过创建大量独立账号来表达每次运行。

### 3.3 账号后端类型

- **本地账号**：由平台内部撮合引擎执行订单，适用于回测与实时模拟盘，也可承载只输出信号的策略运行。
- **平台账号**：由平台适配层连接外部交易平台执行订单，适用于自动化交易。
- 账号的自动交易能力由其绑定的执行后端决定。本地账号通过本地撮合完成自动交易，平台账号通过外部平台执行自动交易。

### 3.4 运行模式与执行方式

- **回测**：历史行情 + 逻辑交易账号 + 本地撮合。
- **实时模拟**：真实行情 + 逻辑交易账号 + 本地撮合。
- **实时信号**：真实行情 + 逻辑交易账号 + 仅输出信号/日志/通知。
- **自动交易**：真实行情 + 逻辑交易账号 + 平台执行后端。

上述模式共用同一套策略、交易环境与运行结果模型。系统通过行情来源、执行后端和运行结果归档规则来表达不同运行形态。
